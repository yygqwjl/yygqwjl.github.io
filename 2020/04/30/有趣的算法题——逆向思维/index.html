有趣的算法题--逆向思维
发表于 2020-04-30| 更新于 2020-06-01|算法

逆向思维

力扣1423. 可获得的最大点数 Medium

题目和code

几张卡牌 排成一行，每张卡牌都有一个对应的点数。点数由整数数组 cardPoints 给出。

每次行动，你可以从行的开头或者末尾拿一张卡牌，最终你必须正好拿 k 张卡牌。

你的点数就是你拿到手中的所有卡牌的点数之和。

给你一个整数数组 cardPoints 和整数 k，请你返回可以获得的最大点数。

class Solution:
def maxScore(self, cardPoints: List[int], k: int) -> int:
    if len(cardPoints) == k:
        return sum(cardPoints)
    res_opposite = sum(cardPoints[:len(cardPoints) - k])
    current_sum = res_opposite
    left_idx, right_idx = 0, len(cardPoints) - k - 1
    for _ in range(k):
        left_idx += 1 
        right_idx += 1
        current_sum = current_sum - cardPoints[left_idx - 1] + cardPoints[right_idx]  
        res_opposite = min(res_opposite, current_sum)
    return sum(cardPoints) - res_opposite

题解

“题外话”
自己做这道题的时候以为是博弈论dp，看了题解之后原来就是逆向思考之后滑动窗口求最小值。
最终拿k张卡牌，最后会剩下m = len(cardpoints) - k张卡牌，这m张卡牌是连续的，这样就可以使用滑动窗口去算了，求的是可以获得的最大点数，实际上我们求出滑动窗口的最小点数后，用总值减去滑动窗口的最小点数就是可以获得的最大点数。

滑动窗口模板

求长度为k的窗口在len(alist) == m 滑动产生的窗口

left,right = 0,k-1
nowresult = alist[:k]
result = [nowresult]
for i in range(m-k):
    left += 1
    right += 1
    nowresult = alist[left:right+1]
    result.append(nowresult)
print(result) #打印出所有的窗口

因为这道题是算法题，需要优化时间复杂度，就不用sum进行求和，sum求和会超时，所以用 current_sum = current_sum - cardPoints[left_idx - 1] + cardPoints[right_idx] 进行每个窗口值的统计
